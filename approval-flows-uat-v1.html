Approval Flows UAT

Testing Guide

Introduction

This document provides step-by-step testing scenarios for the Approval Flows module in Bayzat. Approval Flows is a routing configuration system that automatically directs employee requests to the appropriate approvers based on defined criteria. Instead of manually assigning approvers, you configure intelligent approval paths that automatically route requests based on employee attributes, request amounts, departments, or custom conditions. Tests should be executed in your designated environment (staging or production as applicable).

The system supports 40+ request types with dedicated configuration tabs for Leave, Expenses, Loans, Air Tickets, Attendance Regularization, and more. Administrators configure advanced (conditional) flows for exception-based routing and default flows for standard cases. Approval flows support multi-step hierarchies, parallel approvers, and role-based routing. This guide focuses on core workflows: accessing approval flows, creating and editing advanced flows, configuring default flows, and testing automatic request routing.

Scenario Steps Expected Result ✓ Help Article

2. Amount: $8,000 (matches flow criteria "Amount > 5000")
3. Department: Finance
4. Submit request
5. Check routing		 Request submitted successfully. System evaluates advanced flows in priority order. First matching flow applied. Request routed to configured approvers for that advanced flow (e.g., Finance Manager). Approver receives notification. Routing automatic. Matching flow
Submit Request Using Default Flow 1. As employee, submit expense
2. Amount: $2,000 (no advanced flow matches)
3. Department: HR
4. Submit request
5. Check routing to default		 Request submitted. No advanced flow criteria match. Default flow applies automatically as fallback. Request routed to default approvers (e.g., Line Manager). Default flow acts as safety net. Ensures all requests have routing path. Default routing
Multi-Step Approval Routing 1. Configure flow with 2 steps
2. Step 1: Department Manager
3. Step 2: Finance Manager
4. Submit request matching flow
5. Verify sequential routing		 Request routes to Step 1 approver first. After Step 1 approval, routes to Step 2 approver automatically. Each step executed sequentially. Sequential steps ensure all approvers approve before completion. System advances through steps correctly. Multi-step approval
Parallel Approvers Within Step 1. Configure flow with 1 step
2. Add 2 approvers to step (Finance Manager, Department Manager)
3. Submit matching request
4. Both approvers notified
5. Approval from any one sufficient		 Both approvers receive request simultaneously (parallel routing). Either approver can approve request (OR logic). If one approver rejects, request is rejected immediately. Parallel approvers reduce bottlenecks when key person unavailable. System distributes correctly. Parallel approvers
Request Rejection Stops Process 1. Request in approval at Step 1
2. Step 1 approver rejects request
3. Check request status
4. Does NOT proceed to Step 2
5. Employee notified of rejection		 Request rejected at any step stops entire approval process immediately. Does not advance to subsequent steps. Employee notified of rejection reason. Request marked as rejected. Cannot be re-evaluated or advanced by other approvers. Rejection is final. Rejection handling
